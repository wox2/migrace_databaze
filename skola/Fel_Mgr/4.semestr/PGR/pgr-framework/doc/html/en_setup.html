Nastavení pro ostatní studia a platformy je obdobné, pokud budete psát aplikace čistě, budou zdrojové kódy přenositelné minimálně i na Linux a Mac. Zdrojové kódy pgr-framework jsou testované pod Windows + VS a Linux + gcc.

PGR Framework a struktura projektových adresářů

PGR Framework je základní sada knihoven pro práci s OpenGL (Win).
Framework obsahuje hlavičkové soubory, .lib a .dll soubory k následujícím knihovnám:

  • FREEGLUT - The OpenGL Utility Toolkit - http://freeglut.sourceforge.net/
  • GLEW - The OpenGL Extension Wrangler Library - http://glew.sourceforge.net/
  • Assimp - Open Asset Import Library - http://assimp.sourceforge.net/
  • DevIL - Image Library - http://openil.sourceforge.net/
  • pgr - PGR knihovna - obsahuje pár užitečných funkcí a závislosti na další knihovny

PGR framework rozbalte do zvolené složky, v návodu používáme h:\pgr.

Nastavení systémových proměnných

Pro snadný přenos projektu mezi uživateli používáme systemové proměnné. Cestu k rozbalenému frameworku uložíme do proměnné PGR_FRAMEWORK_ROOT.

Pro správný běh programů je potřeba přidat cestu k .dll souborum pgr-frameworku do systemove proměnné PATH.

Rychlý přístup do proměnných prostředí přes vyhledávání ve Win 7.

Nebo postupem: klik pravým tlačítkem na ikoně Počítač -> Vlastnosti -> Upřesnit nastavení systému -> karta Upřesnit ->Proměnné prostředí

 

Vytvoříme celkem dvě proměnné v sekci Uživatelské proměnné

Nastavujeme proměnnou PGR_FRAMEWORK_ROOT

Nastavujeme proměnnou PATH

 

Založení projektu ve Visual studiu 2010

Po prvním spuštění Vás VS vyzve k nastavení výchozího prostředí, zvolíme C++.

Projekty vytváříme jako prázdné konzolové C/C++ aplikace.

Nastavení cest pro PGR framework

Pokud tam volba C/C++ není, přidejte nejdříve do projektu nějaké C/C++ soubory.

Shrnutí všech nastavení

  • Zvolit adresář pro projekty - např. H:\pgr
    • Do tohoto adresáře rozbalit podadresář pgr-framework
    • Nastavit systémovou proměnnou PGR_FRAMEWORK_ROOT a PATH
  • V nastaveni projektu ve VisualStudio 2010 (nahore zvoleno All Configurations) přidat:
    • C++ -> General -> Additional Include Directories: $(PGR_FRAMEWORK_ROOT)include
    • C++ -> Preprocessor Definitions: přidat GLEW_STATIC;
    • Linker -> General -> Additional Include Directories: $(PGR_FRAMEWORK_ROOT)lib
    • Linker -> Input -> Additional Dependencies: přidat v Debug konfiguraci: pgrd.lib;, v Release konfiguraci přidat: pgr.lib;
    • Configuration Properties -> General -> Character Set -> Not Set (zrusi podporu UNICODE) - Kvůli problémům se jmény souborů v knihovne DevIL na některých systémech.
